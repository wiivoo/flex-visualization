## Interaction Contracts

### Process View Toggle Button
- Location: Chart header row, right-aligned alongside the chart mode controls
- Label: "Process" (noun, no verb — consistent with "Fleet", "Intraday" toggle labels)
- State: inactive = `text-gray-400 hover:text-gray-600`; active = `bg-white text-sky-700 shadow-sm` within a pill group wrapper
- Entering process view: normal chart unmounts, ProcessViewChart mounts in the same container div — no layout shift
- Exiting process view (pressing "Process" again or a dedicated "X"): normal chart remounts, last selected date/scenario preserved

### Stage Navigation
- Default stage on open: "Forecast" (stage 0) — always start from beginning
- Clicking a stage dot: jump directly to that stage
- Prev/Next buttons: step through stages sequentially
- Disabled state: "Intraday" stage button is visually disabled (`opacity-40 cursor-not-allowed`) and shows tooltip "Intraday data unavailable for this date" when `intradayFull.length === 0`

### Scenario Selection
- Default: "Realistic" — most meaningful for new users (not the "perfect world" default)
- Changing scenario: immediately re-runs optimization with perturbed inputs (debounce 0ms — pure function call, fast enough)
- Both chart overlays AND WaterfallCard update simultaneously on scenario change

### Waterfall Card
- Bars animate from 0 to final value on first render (height transition, 200ms ease-out)
- Switching single EV ↔ fleet via the existing Step2 toggle: WaterfallCard re-renders with new values — no separate toggle inside WaterfallCard itself
- Negative drag bars (DA error, availability error, intraday cost): red fill `bg-red-400` equivalent
- Positive total bars (Perfect, Realized): emerald fill `bg-emerald-500` equivalent
- Fleet mode: adds a second series (lighter blue) alongside the single-EV series within the same chart

## Stage-Specific Visual Progression

The chart must visually communicate information availability at each stage:

### Stage 1 — Forecast
- DA price bars: dimmed (`opacity-40`) — prices not yet known
- Yellow uncertainty corridor (ReferenceArea) spans the overnight plug-in window: communicates "we're guessing"
- Charging blocks shown at estimated positions with dashed borders
- Header badge: "D-2 to D-1 12:00" in `text-[10px] text-amber-600`

### Stage 2 — DA Nomination
- DA price bars: full opacity — auction results revealed
- Uncertainty corridor removed; charging blocks solidify to optimized positions (emerald ReferenceArea)
- Forecast error band removed; replaced by actual price line
- Header badge: "D-1 12:00" in `text-[10px] text-emerald-600`

### Stage 3 — Intraday Adjustment
- Intraday price line visible (sky-blue, same as existing ID3 overlay) when `intradayFull` available
- Forced re-trade zones: red ReferenceArea where car availability differed from forecast
- Re-optimized blocks: emerald, shifted from DA positions
- Header badge: "Day D" in `text-[10px] text-sky-600`
- If `intradayFull` unavailable: gray message strip "Intraday data unavailable — showing DA result only"

## Fleet Mode Visual Contract

When `showFleet === true` in Step2 (process view inherits this state):

- Flex band overlays: semi-transparent blue `fill="#DBEAFE"` `fillOpacity={0.3}` spanning greedy/lazy bounds per slot
- Flex band upper bound label: `text-[10px] text-blue-500` "Max"
- Flex band lower bound label: `text-[10px] text-blue-500` "Min"
- Waterfall: two bars per bucket — "Single EV" (gray-400 fill) and "Fleet/car" (blue-400 fill) — fleet bar always shorter than single-EV drag bars
- Portfolio effect note below chart: `text-[10px] text-blue-600` "√N portfolio effect reduces uncertainty per car"
