**Types:**

```typescript
export type ProcessStage = 'forecast' | 'da_nomination' | 'intraday_adjustment'
export type UncertaintyScenario = 'perfect' | 'realistic' | 'worst'

export const PROCESS_STAGES: { key: ProcessStage; label: string; description: string }[] = [
  { key: 'forecast', label: 'Forecast', description: 'D-2 to D-1 12:00 — estimate availability and energy need' },
  { key: 'da_nomination', label: 'DA Nom.', description: 'D-1 12:00 — day-ahead auction prices revealed, nominate cheapest slots' },
  { key: 'intraday_adjustment', label: 'Intraday', description: 'Day D — actual car availability revealed, re-optimize position' },
]

export const UNCERTAINTY_SCENARIOS: { key: UncertaintyScenario; label: string }[] = [
  { key: 'perfect', label: 'Perfect' },
  { key: 'realistic', label: 'Realistic' },
  { key: 'worst', label: 'Worst case' },
]

export interface StageResult {
  optimizeResult: OptimizeResult
  windowStart: string
  windowEnd: string
  pricesUsed: HourlyPrice[]  // the prices this stage optimized against
}

export interface WaterfallBar {
  label: string
  base: number       // invisible offset (ct/kWh)
  value: number      // visible bar height, negative for drag bars
  color: string      // 'emerald' | 'red' | 'blue'
  isTotal?: boolean
}

export interface ProcessViewResult {
  stages: Record<ProcessStage, StageResult | null>
  waterfall: WaterfallBar[]
  fleetWaterfall: WaterfallBar[] | null
  perfectSavingsCtKwh: number
  realizedSavingsCtKwh: number
  daForecastDragCtKwh: number
  availabilityDragCtKwh: number
  intradayCorrectionCtKwh: number
}
```

**Uncertainty calibration constants** (exposed as named exports so Lars can tune):

```typescript
export const UNCERTAINTY_CONFIG = {
  realistic: { daPriceNoiseEurMwh: 8, plugInVarianceHours: 1, intradaySpreadEurMwh: 3 },
  worst: { daPriceNoiseEurMwh: 20, plugInVarianceHours: 2, intradaySpreadEurMwh: 10 },
} as const
```

**Functions:**

1. `perturbPrices(prices: HourlyPrice[], scenario: UncertaintyScenario, dateSeed: string): HourlyPrice[]`
   - Perfect: return prices unchanged.
   - Realistic/Worst: add Gaussian noise seeded deterministically from `dateSeed` string (use a simple seeded PRNG — multiply-with-carry or similar, no external lib). Noise magnitude from `UNCERTAINTY_CONFIG`.
   - Return new array, do not mutate input.

2. `perturbWindow(windowStart: string, scenario: UncertaintyScenario, dateSeed: string): string`
   - Perfect: return windowStart unchanged.
   - Realistic: shift by ±1 hour (deterministic from seed). Worst: shift by +2 hours (car arrives late).
   - Clamp result to valid hours (14-23 range).
   - Return "HH:MM" string.

3. `computeProcessViewResults(params: { prices: HourlyPrice[], intradayPrices: HourlyPrice[] | null, scenario: ChargingScenario, uncertaintyScenario: UncertaintyScenario, showFleet: boolean, fleetConfig: FleetConfig | null, dateSeed: string }): ProcessViewResult`
   - **Forecast stage:** Call `runOptimization()` with perturbed prices AND perturbed window. This represents "what we thought would happen" with forecast error.
   - **DA Nomination stage:** Call `runOptimization()` with real DA prices but still perturbed window (car hasn't arrived yet). This shows "we got real prices but still guessing availability."
   - **Intraday stage:** If `intradayPrices` available, call `runOptimization()` with intraday prices and real window (car has arrived). If unavailable, return `null` for this stage.
   - Compute waterfall bars:
     - "Perfect" bar: `savings_eur` from optimization with real prices + real window, base=0, color='emerald', isTotal=true
     - "DA Error" bar: difference between perfect and DA-stage savings, negative value, color='red'
     - "Avail. Error" bar: difference between DA-stage and forecast-stage savings due to window perturbation, negative value, color='red'
     - "ID Cost" bar: intraday correction cost (if intraday available), negative value, color='red'
     - "Realized" bar: final realized savings after all drags, base=sum of above, color='emerald', isTotal=true
   - Convert all waterfall values to ct/kWh (divide EUR by energy_charged_kwh).
   - For fleet mode: scale perturbation noise by `1 / Math.sqrt(effectiveFleetSize)` where `effectiveFleetSize = fleetConfig.fleetSize * Math.min(1, (fleetConfig.plugInsPerWeek ?? 3) / 7)`. Compute a second `fleetWaterfall` array with reduced drag bars (per D-12).
   - Build the `OptimizeInput` from the `ChargingScenario` props using: `battery_kwh` from `VEHICLE_PRESETS` lookup by `scenario.vehicleId`, `charge_power_kw` from `scenario.chargePowerKw`, window from `scenario.plugInTime`/`scenario.departureTime` formatted as "HH:00", `base_price_ct_kwh: 5.0`, `margin_ct_kwh: 2.15`, `customer_discount_ct_kwh: 1.0`.

**Seeded PRNG:** Implement a simple deterministic RNG seeded from a string (hash the date string to a number, use multiply-with-carry). This ensures `useMemo` stability — same date + same scenario = same perturbation. Do NOT use `Math.random()`.
