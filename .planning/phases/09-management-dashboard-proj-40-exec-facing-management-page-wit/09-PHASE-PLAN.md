# Phase 9 — Management Dashboard (PROJ-40)

Build an exec-facing `/management` dashboard that surfaces YoY flex-value, monthly savings totals, and an auditable explainer (reconciliation equation + avg QH profile) using a fixed scenario configurable via a shadcn `Sheet` drawer. Data is precomputed monthly from `public/data/smard-prices-qh.json` and served as static JSON for zero runtime cost.

## Authoritative spec

`features/PROJ-40-management-dashboard.md` — full UI-SPEC, KPI tiles, YoY panel, explainer, settings drawer, data flow, MGMT-01..MGMT-10 requirements.

## Plan Summary

### 09-01 — Data Foundations (Wave 1)
- `src/lib/management-config.ts` — types (`ManagementScenario`, `MonthlyAggregate`, `YoyDatum`, `ExplainerData`), default fixed scenario (battery 60 kWh, charge 7 kW, plug-in 18:00, departure 06:00, 4 sessions/week), localStorage key `flexmon-management-scenario-v1`.
- `src/lib/management-helpers.ts` — pure functions: `aggregateMonthly(qhPrices, scenario)`, `computeYoy(monthly, yearA, yearB)`, `computeExplainer(qhPrices, month, scenario)`, `reconcile(spread, energyPerSession, sessions)`.
- `scripts/precompute-management-monthly.mjs` — reads `public/data/smard-prices-qh.json`, emits `public/data/management-monthly.json` with `{ generatedAt, schemaVersion: 1, scenario: DEFAULT, monthly: [...], explainer: {...} }`.
- `.github/workflows/update-smard-data.yml` — add `node scripts/precompute-management-monthly.mjs` step after existing SMARD update and include `public/data/management-monthly.json` in the commit.
- Requirements: MGMT-10, (foundation for MGMT-02/03/06).

### 09-02 — KpiTile + YoyBarChart (Wave 2)
- `src/components/management/KpiTile.tsx` — card w/ big number (`tabular-nums`, `text-4xl`), label, Δ vs prior period (green/red), inline SVG sparkline (12 points).
- `src/components/management/YoyBarChart.tsx` — Recharts `ComposedChart` with two `<Bar>` series (2025 brand red `#EA1C0A`, 2026 gray `#9CA3AF`), Δ% LabelList above 2026 bars, CartesianGrid, `tabular-nums` axis ticks.
- Requirements: MGMT-02, MGMT-03.

### 09-03 — ExplainerPanel + SettingsDrawer (Wave 2)
- `src/components/management/ExplainerPanel.tsx` — Recharts `ComposedChart` with `<Line>` for avg QH profile (ct/kWh), `<ReferenceArea>` for charging window + baseline window, reconciliation equation row (`spread × energyPerSession × sessions ≈ monthly`), energy-per-QH tooltip (1.75 kWh @ 7 kW).
- `src/components/management/SettingsDrawer.tsx` — shadcn `Sheet` (install via `npx shadcn@latest add sheet --yes`), inputs for battery kWh, charge kW, plug-in/departure time, sessions/week; persists to localStorage; reset-to-default button.
- Requirements: MGMT-04, MGMT-05, MGMT-06, MGMT-07.

### 09-04 — Page wiring + password gate (Wave 3)
- `src/app/management/page.tsx` — client component, `<Suspense>` wrapper, loads `/data/management-monthly.json`, applies scenario from localStorage, renders time-period toggle (YTD | Last 12mo | All), 4 KPI tiles, YoY bar chart, explainer panel, settings drawer trigger.
- Password gate via shared `flexmon-session` JWT cookie (middleware already intercepts `/v2` pattern — extend matcher or client-side redirect on 401).
- Graceful empty-state when `management-monthly.json` absent (404) or stale.
- Requirements: MGMT-01, MGMT-08, MGMT-09.

## Threat Model (STRIDE — low-risk static dashboard)

| Threat | Category | Component | Disposition | Mitigation |
|--------|----------|-----------|-------------|------------|
| T-09-01 | Information Disclosure | `/management` page | mitigate | Enforce `flexmon-session` JWT via middleware matcher or client-side redirect on 401; same gate as `/v2`. |
| T-09-02 | Tampering | localStorage scenario | accept | Client-only preference; server trusts precomputed JSON, not user input. No PII. |
| T-09-03 | Denial of Service | `/data/management-monthly.json` | accept | Static file served by Vercel CDN; no runtime compute on request. |
| T-09-04 | Spoofing | Precompute workflow | mitigate | Runs in GitHub Actions with pinned Node 20; commits signed via default workflow identity. |
