# Phase 8 — UI Design Contract

## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, chart dot radius |
| sm | 8px | Compact element spacing, table row padding, card inner gap |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Section padding, card header padding |
| xl | 32px | Layout column gaps, split-layout divider spacing |
| 2xl | 48px | Major section breaks (consumer section → management section) |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets on toggle controls: minimum 44px height (browser accessibility minimum)
- Chart margin right: 48px (matches MonthlySavingsCard pattern for right YAxis label clearance)
- Variant picker card: 160px min-width per card in horizontal flex row

## Typography

All numeric displays use `tabular-nums`. Code-like labels (timestamps, prices, slot indices) use `font-mono`. Matches existing /v2 codebase conventions exactly.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Card title | 16px (text-base) | 700 (font-bold) | 1.3 | CardTitle in every Card header |
| Body / descriptor | 11px (text-[11px]) | 400 (font-normal) | 1.6 | Subtitle lines under card titles, parameter labels |
| Micro label | 10px (text-[10px]) | 700 (font-bold) | 1.4 | Section section headers (uppercase + tracking-wider), column headers |
| Data / number | 12px (text-[12px]) | 600 (font-semibold) | 1.4 | Inline data values in tables and breakdown lists |
| Display metric | 20px (text-xl) | 700 (font-bold) | 1.2 | Hero ROI numbers: annual savings, payback period, NPV |
| Small annotation | 9px (text-[9px]) | 600 (font-semibold) | 1.4 | Sub-table section headers (e.g. "Quarterly"), assumption footnotes |

Font weights used: 400 (regular) and 600/700 (semibold/bold). No other weights.

## Color

CSS custom properties via HSL vars (`hsl(var(--...))`) — all existing tokens. Brand color `#EA1C0A` used inline as Tailwind arbitrary value where semantic token does not exist (same as /v2).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` ≈ #FCFCFC | Page background, chart background |
| Secondary (30%) | `hsl(var(--card))` = white + `border-gray-200/80` | All Card surfaces, variant picker cards |
| Accent (10%) | `#EA1C0A` | Selected variant ring, active country toggle, selected date in MiniCalendar, active reg-toggle highlight |
| Savings / optimized | `emerald-*` scale | Battery arbitrage savings, PV self-consumption gain, positive ROI values |
| Warning / unmanaged | `red-*` scale | No-battery baseline cost, negative values, DE export-prohibited indicator |
| Neutral | `gray-*` scale | Labels, secondary text, disabled states, chart grid lines |
| Destructive | `hsl(var(--destructive))` ≈ red-500 | Not used in Phase 8 (no destructive user actions) |

**Accent reserved for:** selected variant card ring, active country button (DE/NL toggle), selected calendar date ring, MiniCalendar month-picker active month background, 2000W scenario toggle active state. Nowhere else.

**Special semantic indicators:**
- DE grid-export prohibited: `bg-gray-100` + `text-gray-400` + strikethrough on discharge-to-grid label in chart legend. Tooltip on hover references "VDE-AR-N 4105:2026-03".
- NL post-2027 regime badge: `bg-amber-50 border-amber-200 text-amber-700` pill badge, text "post-2027 regime", shown adjacent to NL country button when NL active.
- PV self-consumption series: `#F59E0B` (amber-400) — distinct from arbitrage emerald to visually separate the two value streams.
- Battery SoC curve: `#3B82F6` (blue-500) — consistent with the intraday ID3 series color already used in /v2 chart.
- Projected/estimated data: `fillOpacity={0.4}` + `strokeDasharray="4 3"` (matches MonthlySavingsCard projected pattern).
- Assumption footnotes: `text-amber-600` inline asterisk marker.

## Screens and Component Inventory

### Screen 1: Battery Scenario Picker (top of page, above the fold)

**Component: `BatteryVariantPicker`**

Three cards arranged in a horizontal flex row (gap-4) on desktop 1440px. Each card is a shadcn `<Card>` with:
- Product name in `text-base font-bold text-[#313131]`
- Install type badge: "Plug-in" (gray pill) or "Electrician req." (amber pill with ⚠ icon from lucide-react)
- Spec grid (2 columns, 10px font): Capacity / Power / RTE / Price (incl. VAT) / Warranty
- Assumption marker on price fields with LOW confidence (asterisk + footnote below picker)
- Selected state: `ring-2 ring-[#EA1C0A] ring-offset-2 scale-[1.02]` on the card container
- Inactive state: `border-gray-200/80 shadow-sm`

Three variants exactly as specified in CONTEXT.md + RESEARCH.md:
- Variant A: Marstek Venus B — 2.0 kWh, Schuko plug-in, 19% VAT
- Variant B: Anker SOLIX Solarbank 2 E1600 Pro — 1.52 kWh + 800 Wp PV, 0% VAT
- Variant C: Marstek Venus E 3.0 — 4.6 kWh usable, electrician required, 19% VAT

**Country + Tariff Controls (inline with picker)**

Country toggle (DE / NL): segmented control (`bg-gray-100 rounded-full p-0.5`, two `button` elements, active state `bg-white shadow-sm text-[#313131]`, inactive `text-gray-400`). NL active shows amber "post-2027 regime" badge inline.

Tariff dropdown: shadcn `<Select>`. Options differ by country:
- DE: Tibber DE, aWATTar, Rabot Charge, Octopus DE (default: aWATTar)
- NL: Frank Energie, ANWB, Tibber NL, Zonneplan (default: Frank Energie)

Household load input: text input (shadcn `<Input>`) labeled "Annual consumption (kWh)" — default 2500. Inline below country/tariff row. `text-[12px] tabular-nums`.

### Screen 2: Intra-day Battery Visualization

**Component: `BatteryDayChart`**

Single `<ResponsiveContainer width="100%" height={320}>` wrapping a Recharts `<ComposedChart>`.

Chart series (layered in render order, bottom to top):
1. `<Area>` — Household load (kWh per slot): `fill="#E5E7EB"` (gray-200), `fillOpacity={0.5}`, `stroke="#9CA3AF"` (gray-400), `strokeWidth={1}`. YAxis left.
2. `<Area>` — PV generation (kWh per slot, Variant B only): `fill="#FEF3C7"` (amber-100), `fillOpacity={0.6}`, `stroke="#F59E0B"` (amber-400), `strokeWidth={1.5}`. YAxis left. Hidden when `includePv === false`.
3. `<Bar>` — Battery charge/discharge action (kW, positive = charge): `fill="#3B82F6"` (blue-500) for charging slots, `fill="#10B981"` (emerald-500) for discharge-to-load slots. `fillOpacity={0.65}`. YAxis left. DE: discharge bars capped at load line — bars above zero only (never negative, no grid export shown).
4. `<Line>` — Electricity price (ct/kWh): `stroke="#EA1C0A"`, `strokeWidth={2}`, `dot={false}`. YAxis right. `tickFormatter={(v) => v.toFixed(0) + ' ct'}`.
5. `<Line>` — Battery SoC (kWh): `stroke="#3B82F6"` (blue-500), `strokeWidth={1.5}`, `strokeDasharray="4 3"`, `dot={false}`. YAxis right, second right axis (if Recharts supports dual right) — or annotate as a separate scale note.

DE export prohibition rendering: when country is DE, add a `<ReferenceArea>` for any slots where the battery SoC would otherwise suggest grid export; render with `fill="#F3F4F6"` (gray-100) and `fillOpacity={0.8}` over those slots. Chart legend entry: "Grid export (prohibited DE)" with `text-gray-400 line-through`.

XAxis: HH:00 labels, tick interval 4 (every hour for hourly resolution, every 4 for QH). `tick={{ fontSize: 10, fill: '#6B7280' }} tickLine={false} axisLine={false}`.

Tooltip: white rounded card, `border border-gray-200 shadow-lg px-3 py-2 text-[12px] space-y-0.5`. Shows: time slot, price ct/kWh, PV kWh (if applicable), load kWh, battery action kW (charge/discharge/idle), SoC start → end kWh, slot savings EUR.

MiniCalendar: reuse existing `<MiniCalendar compact />` component verbatim for date selection. Positioned above the chart. `requireNextDay={false}` (battery page does not need next-day data; SoC carryover is reset to 0 at day start).

### Screen 3: Consumer ROI Card

**Component: `BatteryRoiCard`**

Mirrors `SessionCostCard` + `MonthlySavingsCard` layout patterns.

Upper half — per-session equivalent (daily arbitrage summary):
- Two columns: "No battery" (red-50/60 bg, red borders) vs "With battery" (emerald-50/60 bg, emerald borders)
- Shows: avg buy price ct/kWh vs avg effective price after battery (net savings per kWh)
- Arbitrage stream and PV stream shown as separate rows within the "With battery" column

Lower half — annual ROI metrics (4 large display numbers in a 2×2 grid):
- Annual savings EUR: `text-xl font-bold text-emerald-700 tabular-nums`
- Payback period (years): `text-xl font-bold text-[#313131] tabular-nums`
- Break-even year: `text-xl font-bold text-[#313131] tabular-nums`
- 10-year NPV EUR: `text-xl font-bold` — `text-emerald-700` if positive, `text-red-600` if negative

Below each metric: `text-[10px] text-gray-400` descriptor line.

Collapsible formula section (same pattern as SessionCostCard): shows hardware cost incl. VAT, annual standby cost, annual savings breakdown by stream, simple payback formula.

VAT note for Variant A and C: inline asterisk + `text-[10px] text-amber-600` footnote "Hardware price includes 19% VAT (standalone battery, no PV)."

### Screen 5: Regulation Panel

**Component: `RegulationPanel`**

Inline collapsible card (not a modal). Open by default on first load, collapsible via chevron toggle.

DE controls (shown when country = DE):
- Feed-in cap toggle: "800W (current)" / "2000W (proposed)" — segmented control, same pill pattern. Active 2000W state: `ring-2 ring-[#EA1C0A]` + amber badge "Proposed — not enacted (Apr 2026)".
- Grid export: locked to "Prohibited" — not a toggle. Rendered as a read-only pill: `bg-gray-100 text-gray-500 border border-gray-200`. Tooltip on hover: "VDE-AR-N 4105:2026-03 — battery discharge to grid not permitted under Steckerspeicher regime."
- §14a applicability: read-only chip "Not applicable (< 4.2 kW)" in `text-gray-400`.
- VAT rate: read-only per variant (shown as part of variant card, not in regulation panel).

NL controls (shown when country = NL):
- Regime: locked to "post-2027" — read-only pill `bg-amber-50 text-amber-700 border border-amber-200`. Tooltip: "Salderingsregeling ends 2027-01-01. Current net-metering (1:1) not modeled."
- Terugleverkosten: two-option toggle: "Dynamic contract (€0/yr)" / "Fixed contract (€150/yr)". Default: dynamic.
- Export compensation: inline number input (shadcn Input), labeled "% of market rate". Default: 50 (minimum floor until 2030). Range 50–115 (Frank Energie at ~115).
- BTW note: `text-[10px] text-amber-600` inline — "21% BTW assumed for standalone battery (Belastingdienst source pending)."

Layout: 2-column responsive grid (single column below 768px). Each control has a `text-[11px] text-gray-400` descriptor line below the control.

## URL State Contract

Every scenario parameter that changes the computation must be reflected in the URL. Mirror `src/app/v2/page.tsx` `useSearchParams` + `router.replace` pattern exactly.

| URL Param | State Field | Default | Type |
|-----------|-------------|---------|------|
| `variant` | `scenario.variantId` | `'schuko-2kwh'` | `'schuko-2kwh' \| 'balcony-pv-1.6kwh' \| 'wall-5kwh'` |
| `country` | `scenario.country` | `'DE'` | `'DE' \| 'NL'` |
| `tariff` | `scenario.tariffId` | `'awattar-de'` | string (supplier slug) |
| `load` | `scenario.annualLoadKwh` | omitted if 2500 | integer |
| `feedin` | `scenario.feedInCapKw` | omitted if 0.8 | `0.8 \| 2.0` |
| `teruglever` | `scenario.terugleverCostEur` | omitted if 0 | integer |
| `export_pct` | `scenario.exportCompensationPct` | omitted if 50 | integer |
| `date` | `scenario.selectedDate` | latest available | YYYY-MM-DD |

Omit params that match their default (same as /v2 pattern — keeps URLs shareable and clean).

## Interaction Contracts

### Variant Selection
- Click on a variant card: updates `scenario.variantId`, triggers re-optimization and re-render of all dependent components. No page reload.
- Keyboard: variant cards are `<button>` elements; focus ring uses `focus-visible:ring-2 focus-visible:ring-[#EA1C0A]`.

### Country Toggle (DE ↔ NL)
- Switching country: updates price source (SMARD → ENTSO-E), swaps regulation panel controls, swaps tariff dropdown options.
- NL activation: immediately shows amber "post-2027 regime" badge. Does NOT show a modal — inline change only.
- Loading state while ENTSO-E fetches: price curve in chart shows `opacity-50` + `animate-pulse` shimmer on the Line series. Other chart series (load, PV, SoC) remain visible from static data.
- ENTSO-E error (503): non-blocking banner at top of BatteryDayChart: `bg-amber-50 border border-amber-200 text-amber-700 text-[11px] rounded px-3 py-2` "NL prices unavailable — ENTSO-E returned an error. Showing last cached data." No fallback to DE automatically (user must switch manually if desired).

### Feed-in Cap Toggle (DE 800W / 2000W)
- Toggle re-runs optimizer client-side (< 5ms). All downstream components update synchronously.
- 2000W active: amber badge shown. No other visual change to chart — the higher cap may produce more charge/discharge slots which are already visible as taller bars.

### Date Selection (MiniCalendar)
- Selecting a date re-fetches prices for that date (if not already in use-prices.ts cache) and re-runs the day optimizer.
- Calendar color coding: spread color for the battery page is the same `spreadColor()` function as existing MiniCalendar — green/yellow/orange/red by 24h price spread (ct/kWh thresholds unchanged: >20, >15, >10, >5 ct → using existing spread buckets).

### Chart Hover / Click-to-Pin
- Hover on chart data point: shows tooltip (described above). Standard Recharts `<Tooltip />` with custom content renderer.
- No click-to-pin mechanic for Phase 8 (deferred; not needed for this page's story).

### Collapsible Sections
- Formula section in BatteryRoiCard: same `<button>` chevron toggle as SessionCostCard. State is local (not URL-synced).
- RegulationPanel: chevron button with `text-[10px] font-bold text-gray-400 uppercase tracking-wider`. Open by default. State is local.

## Assumptions and Confidence Flags

The following UI elements surface low-confidence data. Each must display an asterisk marker (`*`) linked to an inline footnote using `text-[10px] text-amber-600`:

1. Marstek Venus B retail price (~€499–595): marked LOW confidence in RESEARCH.md. Display asterisk on Variant A price in picker card.
2. NL BTW 21% on standalone battery: marked LOW confidence in RESEARCH.md. Display asterisk on Variant A/C NL price.
3. 2000W feed-in cap "not yet enacted": display amber badge on the 2000W toggle option.
4. PVGIS NL yield for Rotterdam: display asterisk on any NL PV savings figures until the PVGIS API call is confirmed at Wave 0 script run time.

Assumption footnotes appear below the variant picker as a `<ul>` of `text-[10px] text-gray-400` lines, each prefixed with `*`.
