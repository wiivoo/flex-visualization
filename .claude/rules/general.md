# General Project Rules

This file is the short pointer for shared repo conventions. Do not duplicate roster or routing rules here.

## Read First

- Product and setup: `README.md`
- Feature inventory: `features/INDEX.md`
- Team roster: `.claude/TEAM.md`
- Task routing and hiring: `.claude/rules/team-orchestration.md`

## Core Conventions

- Read files before modifying them.
- Verify existing routes, components, and APIs instead of guessing.
- Track feature work in `features/INDEX.md` and the matching `features/PROJ-*.md` spec.
- Use commit messages in the form `type(PROJ-X): description`.
- Keep product behavior changes separate from pure docs or workflow changes when possible.
